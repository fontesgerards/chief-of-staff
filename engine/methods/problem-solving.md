# problem-solving — structured (McKinsey 7-step)

> Invoked situationally when the principal is working a hard problem, not on every run. Produces a workplan and a recommendation, not just analysis.

1. **Define the problem** — What key question must we answer? State it as one sharp question with a clear decision-maker and success criteria.
2. **Structure the problem** — What are the key elements? Decompose into a MECE issue tree (mutually exclusive, collectively exhaustive).
3. **Prioritize issues** — Which branches most affect the answer? Prune the tree to the few that matter; drop the rest explicitly.
4. **Develop the workplan** — Where/how to spend time? Per priority issue: hypothesis, analysis needed, data source, owner, end-product.
5. **Conduct analyses** — What are we trying to prove/disprove? Do the work; favor the analysis that would most change the answer.
6. **Synthesize findings** — What do the findings imply? Roll up to so-whats, not just data.
7. **Develop recommendations** — What should we do? A clear, defensible recommendation with the next actions.

Output the recommendation top-down (lead with the answer) — pair with `minto.md`.
