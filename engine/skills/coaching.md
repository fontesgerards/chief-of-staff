# coaching — strengths-based weekly development

> Uses `engine/methods/strengths-coaching.md`. Load `procedural/coaching.md` for the principal's preferred depth (lightweight nudges vs. structured review — see Open Questions in the plan).

## Steps
1. Read `core/identity.md` + `core/current-priorities.md` (role context).
2. Review the week's `episodic/` (meetings, decisions, interactions) for real moments.
3. Apply the strengths-coaching method: anchor on a strength, name one or two specific, role-relevant moves grounded in actual episodes.
4. End with a concrete experiment for the coming week.

## Output
A short, specific coaching note (not a list). Evidence-grounded. No outward action.

## Test scenario (verification)
- A coaching run references real episodes and the principal's role/priorities from `core/`, and applies the strengths-based method.
